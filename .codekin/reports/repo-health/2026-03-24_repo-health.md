A targeted pass to reflect the current message shapes and flow would prevent integration issues for external contributors.

4. **[MEDIUM] Promote ESLint warning-backlog rules to errors incrementally** — 12 rules are intentionally demoted to `warn`. The `no-unsafe-assignment`, `no-unsafe-argument`, `no-unsafe-member-access`, and `no-unsafe-return` family are highest priority because they can mask real runtime type mismatches. Target one or two per sprint with a corresponding cleanup PR.

5. **[MEDIUM] Delete superseded rename branches** — `origin/refactor/shepherd-to-orchestrator` and `origin/fix/remove-shepherd-rename-orchestrator` are far behind `main` and their intent has already landed. Delete both to avoid confusion.

6. **[LOW] Clarify `CONTRIBUTING.md` dev setup** — The `npm install --prefix server` step is correct but the context is unclear (needed for running server tests directly, not needed for `npm run build`). A short note distinguishing "run server tests" vs "build the project" would reduce new-contributor friction.

7. **[LOW] Add `exactOptionalPropertyTypes: true` to tsconfigs** — This catches a class of subtle bugs where `{ prop: T | undefined }` is treated as assignable to `{ prop?: T }`. Safe to add now given the strict baseline already in place. Likely a small number of type fixes needed.

8. **[LOW] Enable GitHub branch auto-delete on merge** — Set "Automatically delete head branches" in repo settings to prevent the accumulated-merged-branches problem from recurring.

9. **[LOW] Investigate 2 dependencies with no `license` field** — These are transitive sub-dependencies of express/ws and are very likely MIT, but confirming via their npm pages or source repos closes the compliance audit cleanly.

10. **[LOW] Consider adding `knip` or `ts-prune` to CI** — TypeScript's `noUnusedLocals` catches internal dead code but not cross-module unused exports. A tool like `knip` would surface exported symbols that are never imported anywhere, providing a higher-confidence dead-code signal as the codebase grows.
