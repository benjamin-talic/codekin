## Recommendations

Ordered by impact:

1. **Update CHANGELOG.md** — The changelog is ~80 commits behind the current `main`. Use the draft above to backfill the v0.3.7 entry and create an `[Unreleased]` section for post-tag changes. This is the largest documentation gap.

2. **Delete 5 merged remote branches** — All 5 non-main remote branches (`feat/repos-path-empty-state`, `fix/code-review-security-fixes`, `fix/diff-toolbar-bg-color`, `fix/repo-health-audit-fixes`, `fix/task-popup-overlap-diff-view`) are fully merged. Clean them up with `git push origin --delete <branch>`.

3. **Remove orphan components** — `src/components/SessionList.tsx` and `src/components/SessionListPanel.tsx` are not imported by any other file. Verify they are superseded by the current sidebar and delete them to reduce codebase noise.

4. **Clean `server/dist/` declaration stubs** — Delete `server/dist/review-config.d.ts`, `server/dist/review-handler.d.ts`, and `server/dist/review-routes.d.ts`. These are stale build artifacts with no source counterpart. Adding `server/dist/` to `.gitignore` (if not already) would prevent future drift.

5. **Promote ESLint `warn` → `error` incrementally** — The 6 `no-unsafe-*` rules and `no-non-null-assertion` are intentionally set to `warn` for incremental adoption. Schedule a cleanup sprint to fix remaining violations and promote these rules to `error` to close the type-safety gap.

6. **Document webhook mount location** — `POST /api/webhooks/github` and `POST /api/webhooks/stepflow` are mounted directly in `ws-server.ts` and bypass standard auth middleware (using HMAC instead). Add a comment in `ws-server.ts` near the mount point and a note in `docs/API-REFERENCE.md` explaining this so future maintainers understand the security model.

7. **Tag v0.3.8 after changelog and branch cleanup** — Once the above housekeeping items are done, cutting a new patch tag will produce a clean release boundary and make future changelog generation easier.
